# Contributing

Thank you for considering contributing! Please follow these guidelines to help us review and accept contributions quickly.

1. File issues for bugs or feature requests before submitting PRs so we can discuss scope.
2. Fork the repo and create a feature branch: `git checkout -b feat/your-feature`.
3. Keep commits small and focused. Use descriptive commit messages.
4. Write tests for new features or bug fixes and ensure existing tests pass.
5. Follow the project's coding standards and style (add linting/formatting details here if applicable).
6. Rebase or merge the latest main branch before opening a PR.
7. Open a Pull Request with a clear description of the change, linked issues, and testing notes.

If your change is non-trivial, open an issue first to get feedback on the approach.

Security: Do not include secrets or credentials in PRs. See docs/10-Resources for links to secret-management procedures.

By contributing you agree to license your contributions under the project's license.
